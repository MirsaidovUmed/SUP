Название проекта: "Система управления командами (SUP)"

Описание:
Система управления командами (SUP) - это как компьютерная помощница для эффективной работы в команде.
Она помогает нам управлять всеми нашими проектами и задачами в компании.
Как будто у нас есть свой умный помощник, который помогает нам запланировать, отслеживать и контролировать все наши дела.

Управление Проектами:
Мы можем создавать и называть наши проекты, добавлять описания и указывать,
кто за них отвечает и когда они должны быть завершены.

Работа с Задачами:
Каждый проект разбивается на маленькие задачи.
Например, если у нас есть проект по созданию нового продукта,
то задачи могут быть связаны с дизайном, программированием и тестированием.

Управление Командой:
Мы можем приглашать участников в наши проекты и назначать им роли.
Например, кто-то может быть аналитиком, а другой - программистом. Все работают вместе для достижения общей цели.

Планирование и Контроль:
Мы можем легко планировать, когда начать и завершить проект, а также следить за ходом выполнения задач.
Это помогает нам быть организованными и управлять временем.

План развития проекта:
Реализация логики токенов для повышения безопасности.
Доработка и оптимизация работы с ролями и статусами.
Внедрение системы уведомлений для обеспечения коммуникации внутри команды.
Рефакторинг кода и обработка возможных ошибок.
Тестирование системы в различных сценариях использования.
Документирование API для удобства взаимодействия с системой.

Выполнение:
Созданы основные модели, определяющие структуру данных: пользователи, проекты, задачи, роли и статусы.

Работа с пользователями:
Создан репозиторий, который позволяет добавлять данные о пользователях в базу данных.

Создание сервиса для проверки существования пользователя:
Начата работа над сервисом, который проверяет, существует ли пользователь с указанным email.

Реализация сервиса и хендлера для логина:
Разработаны сервис и хендлер для логина пользователей. Теперь система способна аутентифицировать пользователей.

Реализация хеширования для логина:
Переделан сервис регистрации для отправления хеша в БД
Переделан сервис для проверки хеша для логина

Корректировка initRouter:
Внесены изменения в инициализацию роутера для учета новых сервисов и хендлеров.

Работа с проектами:
Создан репозиторий для добавления данных о проектах в базу данных.
Исправлен SQL-код, добавлены таблицы для статусов и ролей, и настроены обращения к ним.
Создан апи который будет показывать данные одного проекта, по его имени.
Создание таблицы с участниками проекта

Улучшение логики токенов:
Изменена логика создания токенов для увеличения безопасности.
Теперь токен использует только идентификатор пользователя.

Создание логики проектов:
Реализована логика создания проектов, включая репозиторий, сервис и хендлер.
Теперь система способна обрабатывать данные о проектах.

Работа с задачами:
Начата работа над связями с задачами. Создан репозиторий, который будет работать с данными о задачах.
Создан апи который будет показывать данные одной задачи, по её заголовку.

Работа с регистрацией:
Обнаружены и исправлены ошибки, связанные с регистрацией пользователей.
